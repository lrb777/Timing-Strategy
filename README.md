# 基于三线反向突破图与蜡烛图形态的标普500择时策略（无监督学习优化）

## 策略概述

以**三线反向突破图**（Three-Line Break Chart）作为趋势过滤器，以**蜡烛图形态**作为入场触发信号，以**转向线**作为出场信号，并引入**无监督学习**对策略参数和市场状态进行优化。

- 择时标的：标普500指数（`SPX`）
- 数据周期：日线
- 交易方向：多头 / 空仓
- 数据来源：yfinance（回测） / MT5（实盘）
- 理论来源：史蒂夫·尼森《日本蜡烛图技术新解》第六章

---

## 策略框架

```
三线图判断大趋势方向
        ↓
蜡烛图形态寻找具体入场点
        ↓
转向线作为止盈出场信号
        ↓
无监督学习完善策略
```

### 模块一：三线反向突破图（趋势过滤）

仅使用收盘价构图，忽略时间轴。每日以今日收盘价与图中最后一根线比较来决定是否画线：

**处于白线（阳线）阶段时：**

| 条件 | 操作 |
|------|------|
| 今日收盘 > 最后一根白线高点 | 画新白线（高点=今日收盘，低点=前白线高点） |
| 今日收盘在最后一根白线高低点之间 | 不画线 |
| 今日收盘 < 最近 N 根连续白线的最低低点 | 画黑色**转向线**（高点=前白线高点，低点=今日收盘） |

**处于黑线（阴线）阶段时：**

| 条件 | 操作 |
|------|------|
| 今日收盘 < 最后一根黑线低点 | 画新黑线（高点=前黑线低点，低点=今日收盘） |
| 今日收盘在最后一根黑线高低点之间 | 不画线 |
| 今日收盘 > 最近 N 根连续黑线的最高高点 | 画白色**转向线**（高点=今日收盘，低点=前黑线低点） |

- 转向线阈值 N 默认为 3（即需突破前 3 根同色线的极值）
- 趋势状态：当前线为白线 = 牛市，黑线 = 熊市

### 模块二：蜡烛图形态（入场信号）

仅在三线图确认牛市时，识别以下多头形态触发买入（收盘识别，次日开盘入场）：

| 形态 | 量化条件 |
|------|---------|
| 锤子线 | 下影线 ≥ 实体×2，上影线 ≤ 实体×0.1 |
| 启明星 | 长黑＋小实体＋长白，第三根收盘超过第一根中点 |
| 多头吞噬 | 白色实体完全吞噬前一黑色实体 |
| 刺穿线 | 白线收盘超过前一黑线实体中点 |

### 模块三：转向线出场

今日收盘价跌破最近 N 根连续白线中的最低低点 → 三线图画出黑色转向线 → 次日开盘平仓

- 转向线本身即趋势反转信号，无需额外止损条件触发
- 黑色转向线高点 = 前一根白线高点，低点 = 今日收盘价

### 模块四：无监督学习优化（待定）

基础策略验证后，对历史市场状态聚类（波动率、趋势强度、三线图线段特征），在不同状态下自适应选择 N 值与形态组合。

## 快速开始

```bash
pip install -r requirements.txt
```

### 获取数据

```bash
# 使用 config.yaml 默认参数
python fetch_data.py

# 指定起始日期
python fetch_data.py --start 20050101

# 指定完整区间
python fetch_data.py --start 20050101 --end 20241231

# 切换数据源（默认 yfinance，实盘切换为 mt5）
python fetch_data.py --source mt5

# 忽略缓存，强制重新拉取最新数据
python fetch_data.py --refresh
```

数据以 CSV 格式缓存至 `data/raw/`，再次运行直接读缓存不重复请求。数据源和时间区间在 `config/config.yaml` 的 `data` 节中配置。

### Docker 环境

```bash
# 构建镜像
docker compose build

# 获取数据（data/ 目录挂载到宿主机，缓存持久化）
docker compose run --rm strategy python fetch_data.py
docker compose run --rm strategy python fetch_data.py --start 20200101 --end 20260101
docker compose run --rm strategy python fetch_data.py --refresh
```

> **注意**：Docker 容器运行于 Linux，`MetaTrader5` 包不可用，数据源只支持 `yfinance`。实盘信号生成需在 Windows 本地环境运行。
